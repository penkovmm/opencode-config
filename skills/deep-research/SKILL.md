# Deep Research — Глубокое исследование

Ты проводишь структурированное исследование по заданной теме. Три фазы, human-in-the-loop.

## Инструменты

| Инструмент | Для чего |
|---|---|
| `web-search_search_web` | Поиск источников по запросу |
| `web-search_fetch_url` | Чтение содержимого найденных страниц |
| `webfetch` | Альтернативное чтение URL (если MCP не справляется) |
| `vision describe` | Анализ изображений/PDF в источниках |

## Фаза 1: План исследования

```
Пользователь: "Исследуй <тема>"
```

**Что сделать:**
1. Разбить тему на 5-12 конкретных исследовательских вопросов
2. Для каждого вопроса указать: что ищем, какие ключевые слова
3. Вывести план в формате:

```markdown
## План исследования: <тема>

| # | Вопрос | Ключевые слова |
|---|--------|---------------|
| 1 | ... | ... |
| 2 | ... | ... |
```

**Стоп!** Показать план пользователю. Ждать подтверждения. Не начинать Фазу 2 без явного согласия.

## Фаза 2: Глубокий поиск

Для каждого вопроса из плана:

```
1. web-search_search_web с ключевыми словами (limit=5)
2. Для каждого найденного результата → web-search_fetch_url
3. Извлечь ключевые факты, цифры, даты, имена
4. Сохранить в структурированном виде
```

**Правила поиска:**
- Использовать разнообразные формулировки запросов
- Искать на русском И английском
- Проверять交叉-источники (если факт в одном источнике — искать подтверждение)
- Фиксировать URL источников
- Если источник недоступен → пометить и двигаться дальше

## Фаза 3: Финальный отчёт

Скомпилировать всё в Markdown-файл `research_<тема>.md`:

```markdown
# <Тема> — Исследование

**Дата**: <сегодня>
**Методология**: веб-поиск по N источникам

---

## Основные выводы
(3-5 ключевых тезисов)

## Детальные результаты

### 1. <Вопрос 1>
- Факт A (источник: URL)
- Факт B (источник: URL)

### 2. <Вопрос 2>
...

## Источники
1. [Название](URL) — релевантность: высокая/средняя
2. ...

## Ограничения исследования
- ...
```

**Правила отчёта:**
- Каждый факт — с ссылкой на источник
- Отделять факты от мнений
- Указывать даты публикации источников
- Если источники противоречат друг другу — отметить это
- Включить раздел «ограничения»: что не удалось найти

## Ограничения
- Максимум 12 исследовательских вопросов за сессию
- Максимум 5 источников на вопрос (limit=5)
- Не использовать Exa (OPENCODE_ENABLE_EXA не нужен)
- Если web-search MCP недоступен → использовать webfetch с прямыми URL
