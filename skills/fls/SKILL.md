# Файловый сервер fls.penkovmm.ru (filebrowser, чтение и запись)

Доступ к файлам пользователя через HTTP API filebrowser. Полный доступ — чтение, создание,
переименование, перемещение, копирование, удаление, загрузка.

Учётные данные лежат в `~/.fls_creds` (права 600), скрипт читает их сам — пароль через модель
не проходит. Зависимостей нет (только стандартная библиотека Python 3). Токен кэшируется в
`~/.cache/fls_token` (~1.5 ч), поэтому повторные вызовы не логинятся заново.

**Охват:** учётка видит корень filebrowser (на сервере это `/media/hdd/filebrowser/data`) —
рабочие документы пользователя, а не медиатеку на 14 ТБ. Если нужен весь диск NAS — это уже не
сюда, а `ssh nas` (полный доступ к `/media/hdd` и `/media/raid`).

## Использование

Путь к скрипту фиксированный:

```bash
python3 "$HOME/.claude/skills/fls/fls.py" <команда>
```

### Чтение

- `ls [путь]` — листинг каталога (папки сверху, затем файлы с размером и датой). Без аргумента — корень.
- `tree [путь] [--depth N]` — рекурсивное дерево (по умолчанию глубина 2).
- `cat <путь>` — вывести текстовый файл (yaml/json/md/txt/csv/код). Для бинарных подскажет `get`.
- `get <путь> [куда]` — скачать файл локально и напечатать абсолютный локальный путь.
  По умолчанию в `/tmp/fls/`. **Ключевая команда для бинарных документов.**
- `search <запрос> [путь]` — поиск по имени по всему дереву (или внутри указанной папки). Кириллица ок.

### Запись

- `mkdir <путь>` — создать папку.
- `rm <путь>` — удалить файл или папку.
- `mv <src> <dst>` — переименовать или переместить (скачивает, загружает на новое место, удаляет исходник).
- `cp <src> <dst>` — скопировать (скачивает и загружает на новое место).
- `upload <локальный_файл> <путь_на_сервер>` — загрузить локальный файл на сервер.

Пути с пробелами и кириллицей передавай как есть, в кавычках: `ls "25. Материалы ГПН"`.

## Как выбирать команду

- «Что лежит на сервере / в папке X» → `ls` (или `tree` для обзора вложенности).
- «Найди файл про Y» → `search Y`, затем `ls`/`cat`/`get` по найденному пути.
- «Прочитай / покажи содержимое текстового файла» → `cat <путь>`.
- «Прочитай этот PDF / docx / таблицу / посмотри картинку» → сначала `get <путь>`, затем работай
  с локальной копией (см. ниже). Сначала сужай через `ls`/`search`, потом тяни конкретный файл —
  не выкачивай всё подряд.
- «Сохрани файл на сервер» → `upload <локальный> <путь_на_сервер>`.
- «Создай папку» → `mkdir <путь>`.
- «Удали файл/папку» → `rm <путь>`.
- «Переименуй / перемести» → `mv <src> <dst>`.
- «Скопируй» → `cp <src> <dst>`.

## Чтение скачанных документов

`get` возвращает локальный путь. Дальше:

- **PDF, PNG/JPG** → читай напрямую инструментом Read (он понимает PDF и изображения).
- **`.docx`** → конвертируй в текст: `textutil -convert txt -stdout "<локальный путь>"` (macOS).
- **`.xlsx`** → быстрый текст: `python3 -c "..."` через `openpyxl`, либо `textutil` не возьмёт —
  если нет openpyxl, сообщи пользователю.
- **`.pptx`** → текст вытаскивается распаковкой (`unzip -p file.pptx 'ppt/slides/*.xml'` + чистка тегов).

## Заметки

- Если скрипт сообщает об ошибке входа — проверь `~/.fls_creds` (логин/пароль) и доступность
  `https://fls.penkovmm.ru`.
- mv/cp работают через скачивание → загрузку (PATCH заблокирован nginx-прокси). На больших файлах
  могут быть медленными.
- Сервер — filebrowser v2.60.0 на VM `nas` (172.23.79.3:8080), наружу через nginx-proxy.
  Подробности инфраструктуры — в `~/homelab/audit.md`.
