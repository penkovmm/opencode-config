# Research — План исследования

## Триггер
`/research <тема>` или «исследуй <тема>»

## Алгоритм

### Шаг 1: Первичный фреймворк из знаний модели
На основе темы сгенерировать из знаний модели:
- **Список объектов исследования** (5-15 позиций)
- **Структура полей** для каждого объекта: название, описание, уровень детализации (brief/moderate/detailed)

Вывести пользователю. Спросить через AskUserQuestion:
- Добавить/убрать объекты?
- Добавить/убрать поля?
- Временной диапазон поиска? (6 мес / с 2024 / без ограничений)

### Шаг 2: Веб-поиск для дополнения
Запустить web-researcher (subagent через Task) с промптом:
```
## Задача
Тема: {topic}
Текущая дата: {date}
На основе существующего фреймворка найди упущенные объекты и поля.

## Существующий фреймворк
{step1_output}

## Цели
1. Проверить, не упущены ли важные объекты
2. Найти новые объекты по теме за {time_range}
3. Предложить дополнительные поля для сбора данных

## Формат вывода
### Дополнительные объекты
- Название: краткое обоснование
### Дополнительные поля
- Название: описание (зачем нужно)
### Источники
- [Название](URL)
```

**Источники для поиска (web-researcher):**
- Русскоязычные: РБК, Ведомости, Habr, vc.ru, TAdviser, CNews
- Англоязычные: Reddit, Hacker News, GitHub, Dev.to, Medium
- Академические: Google Scholar, arXiv
- Официальные: сайты компаний, пресс-релизы

### Шаг 3: Генерация outline.yaml и fields.yaml
Объединить результаты Шага 1 и Шага 2. Создать файлы:

**outline.yaml** — объекты + настройки выполнения:
```yaml
topic: "Тема исследования"
date: "YYYY-MM-DD"
items:
  - name: "Объект 1"
    category: "Категория"
    description: "Описание"
  - name: "Объект 2"
    ...
execution:
  batch_size: 3
  items_per_agent: 1
  output_dir: "./results"
```

**fields.yaml** — определения полей:
```yaml
fields:
  - name: "название"
    description: "описание поля"
    detail_level: brief  # brief | moderate | detailed
  - name: "дата_выхода"
    description: "дата выхода продукта/события"
    detail_level: brief
```

### Шаг 4: Сохранить и подтвердить
- Создать директорию `./{topic_slug}/`
- Сохранить `outline.yaml` и `fields.yaml`
- Показать пользователю. Ждать подтверждения перед `/research-deep`.
