# Research Deep — Глубокий поиск

## Триггер
`/research-deep` (в директории с `outline.yaml`)

## Алгоритм

### Шаг 1: Авто-поиск outline
Найти `*/outline.yaml` в текущей директории. Прочитать:
- `topic` — тема
- `items` — список объектов
- `execution.batch_size` — параллельных агентов за раз
- `execution.items_per_agent` — объектов на агента
- `execution.output_dir` — куда сохранять (по умолчанию `./results`)

### Шаг 2: Проверка resume
Проверить `{output_dir}/` — какие JSON уже есть. Пропустить已完成 объекты.

### Шаг 3: Запуск агентов (батчами)
Для каждого объекта запустить subagent через Task с промптом:

```
## Задача
Исследуй: {item_name} ({item_description})
Выведи структурированный JSON в {output_dir}/{item_name_slug}.json

## Определения полей (из fields.yaml)
{fields_content}

## Требования к выводу
1. JSON строго по полям из fields.yaml
2. Неуверенные значения помечать "[uncertain]"
3. В конце JSON — массив uncertain с именами неподтверждённых полей
4. Каждый факт — с источником (URL)
5. Все значения на русском (названия можно на английском)

## Формат вывода
{
  "item": "название",
  "fields": {
    "поле1": "значение",
    "поле2": "значение [uncertain]"
  },
  "sources": ["URL1", "URL2"],
  "uncertain": ["поле2"],
  "searched_at": "YYYY-MM-DD"
}

## Стратегия поиска
1. Сгенерировать 5-10 вариаций поискового запроса
2. Искать на русском И английском
3. web-search_search_web → выбрать релевантные URL
4. web-search_fetch_url → извлечь факты
5. Перекрёстная проверка: факт из одного источника → искать подтверждение
```

### Шаг 4: Батчи
- Запускать `batch_size` агентов параллельно
- Ждать завершения батча
- Показывать прогресс: «Готово 5/15»
- Спрашивать подтверждение перед следующим батчем

### Шаг 5: Сводка
После завершения всех объектов:
- Количество завершённых
- Количество с [uncertain]-полями
- Путь к результатам

## Правила
- **Максимум 5 параллельных агентов** (batch_size)
- **Не использовать Exa** — только web-search MCP
- При ошибке агента — пометить объект как failed, продолжить
- Resume: не перезаписывать已完成 JSON-файлы
- Все источники — с URL
